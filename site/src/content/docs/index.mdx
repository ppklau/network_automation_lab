## What you inherit

You step into ACME Investments as the engineer responsible for transforming a manually operated financial network into a fully automated, auditable, intent-driven infrastructure.

The environment is already running. Your job is to operate it — and to understand, at every step, *why* it works the way it does.

<CardGrid>
  <Card title="Multi-region BGP" icon="random">
    London DC1 is fully instantiated. New York, Singapore, and Frankfurt run as stub border routers — the same pattern you'd use in a real migration.
  </Card>
  <Card title="Regulatory constraints" icon="approve-check-circle">
    MiFID II zone isolation, FCA audit trails, and BaFin data residency rules are baked into the automation — not documented as an afterthought.
  </Card>
  <Card title="End-to-end pipeline" icon="rocket">
    GitLab CI/CD, Batfish pre-merge validation, Ansible config push, and Prometheus/Grafana monitoring — all wired together and running.
  </Card>
  <Card title="Three reading tracks" icon="open-book">

## About the companion handbook

This lab is the hands-on companion to the [Network Automation Handbook](https://ppklau.github.io/network_automation_handbook/). The Handbook explains **what and why**. This guide shows you **how** — in a working environment you can run locally.

Every exercise references the relevant Handbook chapter so the two reinforce each other.
