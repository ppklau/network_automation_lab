## Security zones

Every interface on every device belongs to a zone. Zone membership determines reachability — enforced at `fw-lon-01`, not just documented in policy.

| Zone | VLAN | Subnet (DC1) | Description |
|------|------|--------------|-------------|
| TRADING | 100 | 10.1.1.0/24 | Algorithmic trading systems. MiFID II isolated. |
| CORPORATE | 200 | 10.1.2.0/22 | Internal users, email, shared services. |
| DMZ | 300 | 10.1.3.0/24 | Internet-facing services, API gateways. |
| MGMT | 900 | 10.1.0.128/25 | Out-of-band device management only. |

<Aside type="caution">
  TRADING → CORPORATE is an explicit deny at the firewall. The Batfish assertion `assert_no_trading_to_corporate_path` verifies this at every pipeline run. If someone adds a route or ACL that creates a path, the pipeline fails before the config reaches a device.
</Aside>
